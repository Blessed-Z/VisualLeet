# 📖 基础百科：虚拟头节点 (Dummy Node)

## 💡 为什么要用 Dummy Node？
在处理链表问题（特别是需要创建新链表或删除节点）时，最头疼的就是“第一个节点怎么处理”。
- 如果没有 Dummy，你需要单独写 `if` 判断第一个节点是谁。
- 有了 Dummy：
  - 它就像一个**“临时挂钩”**。
  - 所有的操作逻辑都可以统一化（因为每个真正的节点都有前驱节点了）。
  - 最后只要 `return dummy.next` 就能拿到结果。

## 📋 变量小字典
- **`dummy`**：虚拟领队，固定不动。
- **`curr`**：施工队长，负责给新队伍接人。
- **`list1 / list2`**：两条原始队伍的领头人。

## 📈 进阶思考
这道题能用递归做吗？
- 可以！递归的思路是：`比较 -> 选小的 -> 让剩下的去递归合并`。递归法代码极短，但会占用 $O(N+M)$ 的栈空间。对于链表合并，迭代法（本题做法）通常更省内存。
