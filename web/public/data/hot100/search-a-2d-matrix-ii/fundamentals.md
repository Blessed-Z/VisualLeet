# 📖 基础百科：二维有序搜索

## 💡 为什么叫“抽象二叉搜索树”？
如果你把右上角的那个点拎起来，让它当“树根”：
- 它的左边邻居（较小）就像是**左子树**。
- 它的下边邻居（较大）就像是**右子树**。
所以，在这个有序矩阵里搜索，本质上就是在玩一次**二叉树搜索**！

## 📋 变量小字典
- **`matrix[i][j]`**：你现在踩着的那个格子。
- **`target`**：你要找的那个宝藏数字。
- **`i` (row)**：行索引，控制上下。
- **`j` (col)**：列索引，控制左右。

---

## 📈 进阶思考
如果矩阵只有**行**有序，或者只有**列**有序，这个方法还灵吗？
- 不灵了。这个算法之所以厉害，是因为它同时利用了横向和纵向的有序性。如果只有单向有序，我们可能就需要对每一行分别做**二分查找**（复杂度会变成 $O(M \log N)$）。
