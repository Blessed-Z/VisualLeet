# 📚 矩阵查找基础百科

### 🌟 核心概念：杨氏矩阵查找 (Young Tableau Search)
虽然这道题是一个完全有序的数组弯曲成的矩阵，但它符合“每一行有序且每一列有序”的特性。在这种矩阵中查找，最经典的方法就是从右上角（或左下角）出发。

---

### 📖 变量小字典：

*   **`m`**: 矩阵有多少行。
*   **`n`**: 矩阵有多少列。
*   **`i`**: 我们的搜寻员当前的行位置（初始为 0）。
*   **`j`**: 我们的搜寻员当前的列位置（初始为 n-1）。
*   **`matrix[i][j]`**: 当前搜寻员脚下的宝箱值。

---

### 🔍 时间复杂度：
为什么是 $O(M+N)$？因为每次比较我们都会要么划掉一行，要么划掉一列。最坏的情况下，我们会从右上角一直走到左下角，总共走过的步数就是行数加上列数。
