# 🧩 括号匹配基础百科

### 🌟 核心概念：为什么栈是括号的“天敌”？

括号匹配就像是剥洋葱，或者剥套娃。最里面的括号最先配对成功。
栈（先进后出）刚好符合这个特性：
*   先进去（最外面）的左括号，最晚被匹配。
*   后进去（最里面）的左括号，最早被匹配。

### 📚 变量小字典

| 变量名 | 比喻 | 作用 |
| :--- | :--- | :--- |
| `stack` | **记事簿/魔法桶** | 记录左括号的索引，或者上一个不匹配的右括号索引。 |
| `ans` | **最高纪录** | 到目前为止，我们看到的最长有效字符串长度。 |
| `i` | **当前坐标** | 当前括号在字符串中的位置。 |
| `stack[-1]` | **地基/参照点** | 离当前位置最近的、无法匹配的点。 |

---

### ⚠️ 敲黑板

*   **连续性**：必须是连续的一串。
*   **哨兵节点**：初始化 `stack = [-1]` 是因为程序员喜欢从 0 开始计数，`-1` 能让第一个位置的计算变得正确。
*   **有效括号串**：意味着每一对 `()` 都能完美合拢，不多不少。
