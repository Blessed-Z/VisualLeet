# 🪜 断掉的阶梯：在旋转数组中找人

小朋友，想象你正在爬一个长长的、按数字从小到大排列的阶梯。但是，突然间阶梯在一个地方断开了，上半截被挪到了下半截的前面！

比如原来的阶梯是：`[0, 1, 2, 4, 5, 6, 7]`
旋转后变成了：`[4, 5, 6, 7, 0, 1, 2]`

现在的任务是：给你一个目标数字 `target`，你要在最短的时间内找到它在第几级台阶。

### 🪄 搜寻秘诀：

哪怕阶梯断了，它依然保持了一个非常重要的特性：**如果你把它从中间切开，至少有一半依然是完好、有序的！**

1. **观察中点 (`mid`)**：
   我们看正中间的那级台阶。
2. **确认哪边是有序的**：
   * 如果 `nums[left] <= nums[mid]`，说明**左半边**是完好的上升阶梯。
   * 否则，说明断点在左边，**右半边**一定是完好的上升阶梯。
3. **判断目标在哪边**：
   * 如果左半边是有序的，我们就看 `target` 是不是在左半边的范围内。在的话，就把搜索范围缩小到左边；不在的话，就去右边（断开的那边）找。
   * 如果右半边是有序的，同理，看 `target` 是否在右半边的范围内。
4. **循环往复**：
   每次我们都能扔掉一半的台阶，直到找到那个人！

### 💡 核心道理：
在旋转数组里，二分查找的关键不是找“大小”，而是先找“哪边是有序的”。只要有一边是有序的，我们就能用范围判断来决定下一步往哪走。
