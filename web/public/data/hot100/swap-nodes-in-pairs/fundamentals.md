# 📖 基础百科：链表的原地交换

## 💡 为什么不直接交换值？
在实际开发中，链表的“值”可能是一个非常庞大的数据对象（比如一整张用户信息图片）。
- **交换值**：需要拷贝大量数据，效率低且浪费内存。
- **交换节点**：只需要改变几个指针的指向（就像改一下收件地址），速度极快且不占额外空间。
所以在算法面试中，通常要求“不修改内部的值”。

## 📋 变量小字典
- **`dummy`**：我们的施工指挥中心。
- **`curr`**：施工队长。
- **`node1 / node2`**：当前正在手拉手的两个交换对象。

## 📈 进阶思考
这道题能用递归做吗？
- 当然可以！递归思路更短：`交换当前这一对 -> 让剩下的去递归交换 -> 把当前对的结尾连上剩下的结果`。
