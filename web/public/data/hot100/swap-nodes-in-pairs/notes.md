# 24. 两两交换链表中的节点 (Pair Swap)

## 💡 深度解析
小朋友，想象有一排小朋友手拉手。我们要玩一个**“双双换位”**的游戏。
每两个挨着的小朋友都要互换位置。

### 核心策略：施工队长的牵手指南
为了不让队伍断开，我们需要一个“施工队长”在旁边盯着。
1. **队长定位**：队长站在要换位的这两个小朋友的前面。
2. **第一步（跨越）**：队长松开原来的手，直接拉住第二个小朋友。
3. **第二步（断路）**：第一个小朋友松开拉着第二个小朋友的手，去拉住后面更远的人。
4. **第三步（转身）**：第二个小朋友转过身，反手拉住刚才在他前面的那个小朋友。
5. **任务达成**：队长往前挪两步，继续盯着下一对。

注意：我们不能把小朋友身上的号码牌（值）撕下来互换，必须让他们本人换位置哦！

---

## 🛠️ 执行步骤
1. 设置 `dummy` 节点。
2. `curr` 初始化在 `dummy`。
3. 循环检查 `curr.next` 和 `curr.next.next` 是否存在。
4. 定义 `node1` 和 `node2` 为下一对。
5. 重新连接三个指针：`curr->node2`, `node1->node2.next`, `node2->node1`。
6. `curr` 移动到 `node1`。

---

## 📈 复杂度分析
- **时间复杂度**：$O(N)$。我们一次处理两个，一共扫一遍链表。
- **空间复杂度**：$O(1)$。只用了几个固定的标记位置。
