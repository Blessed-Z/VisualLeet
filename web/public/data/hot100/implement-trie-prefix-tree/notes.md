# 🌳 魔法字典树：单词的超级收纳盒

小朋友，想象你有一本超级厚的神奇字典。如果把单词一个一个乱丢进去，找的时候会非常慢。
为了找得快，我们发明了一种“树”形收纳法，它叫 **Trie（前缀树）**，发音和“Try”一样。

### 🪄 怎么收纳和寻找？

1.  **种下一棵树（初始化）**：
    一开始，只有一个空空的树根。
2.  **存入单词（Insert）**：
    假如我们要存入单词 `apple`。
    我们从树根出发，看看有没有 `a` 这条路。没有？那就建一条 `a` 的路。
    顺着 `a` 往下，看看有没有 `p` 的路。没有？建一条 `p` 的路...
    走到最后，我们在 `e` 的位置挂上一个**特殊牌子（比如 `#` 号）**，代表“这是一个完整单词的结尾”。
    如果再存入 `app`，我们顺着刚才的路走完 `a-p-p`，直接在第二个 `p` 那里挂个 `#` 号就行了，非常省空间！
3.  **精确查找（Search）**：
    你要找 `apple`，就顺着 `a -> p -> p -> l -> e` 的路走。如果路断了，或者走到最后没看到 `#` 号牌子，说明字典里没这个词。
4.  **前缀探测（StartsWith）**：
    比如你想知道字典里有没有以 `app` 开头的词。你只要顺着路走完 `a -> p -> p`，只要路没断，就算成功！不需要看有没有 `#` 号牌子。

### 💡 为什么它这么快？
不管字典里存了一万个词还是一百万个词，你想找 `apple`，最多只需要走 5 步！它的速度只和**单词的长度**有关，和字典有多厚完全无关。这就是魔法字典树的威力。
