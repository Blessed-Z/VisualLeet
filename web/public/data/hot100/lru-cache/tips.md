# 💡 LRU 缓存：思路巧记

### 🎵 缓存小曲

> **哈希快速找，链表管进出。**

### 🔑 核心关键点

1.  **哈希表 (HashMap)**: 
    *   存储 `key` 到 `Node` 的映射。
    *   实现 O(1) 的查询。
2.  **双向链表 (Doubly Linked List)**: 
    *   维护节点的访问顺序。
    *   **头部 (Head)**: 最近刚访问过的节点。
    *   **尾部 (Tail)**: 最久未访问的节点（即将被删除的）。
3.  **核心操作**:
    *   **Get**: 找到节点，移到头部，返回值。
    *   **Put**: 
        *   若已存在：更新值，移到头部。
        *   若不存在：
            *   若满了：删除尾部节点，删除哈希表记录。
            *   新建节点，插入头部，加入哈希表。
4.  **哨兵节点 (Sentinel Nodes)**:
    *   使用 `dummyHead` 和 `dummyTail` 能避免复杂的空链表判断。
