# 21. 合并两个有序链表 (The Sorting Parade)

## 💡 深度解析
小朋友，想象有两队小朋友在排队进教室，每队小朋友都已经按身高从矮到高排好了。
我们要把这两队合在一起，变成一队**更长的、依然按身高排好**的队伍。

### 核心策略：挑小的先走
我们要准备一个**“虚拟的领队”**（Dummy Node），他后面就是我们要排出的新队。
1. **对比**：看看两队排在最前面的两个小朋友，谁更矮。
2. **入队**：把更矮的那个小朋友拉出来，接到新队伍的末尾。
3. **继续**：刚才被拉走小朋友的那一队，后面的人往前补位，继续和另一队的领头人比。
4. **扫尾**：如果某一队小朋友走光了，另一队剩下的人不用比了，他们已经排好序了，直接整队跟在新队后面就行！

---

## 🛠️ 执行步骤
1. 创建一个 `dummy` 节点。
2. 用 `curr` 指针指向 `dummy`。
3. `while` 循环：比较 `list1.val` 和 `list2.val`：
   - 谁小，`curr.next` 就指向谁，并移动对应的 `list` 指针。
   - `curr` 向后移一步。
4. 循环结束，把没走完的那一队直接挂在 `curr.next`。
5. 返回 `dummy.next`。

---

## 📈 复杂度分析
- **时间复杂度**：$O(N + M)$。我们只需要把每个小朋友都看一遍。
- **空间复杂度**：$O(1)$。我们就多请了一个“虚拟领队”，没开新教室。
