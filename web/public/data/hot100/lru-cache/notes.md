# 146. LRU 缓存：神奇的玩具箱

想象一下，你有一个超级酷的**玩具箱**（缓存），但这个箱子的空间有限，只能放 3 个玩具。

### 🏫 故事背景：喜新厌旧的玩具箱
为了保证箱子里永远是小朋友最喜欢的（最近玩过的）玩具，我们定了一个规则：

1.  **新宠入宫**：如果你拿了一个新玩具放进去，它就是“最新”的，排在最前面。
2.  **旧爱复宠**：如果你从箱子里拿出一个旧玩具玩了一下，那它立刻就变成了“最新”的，也要挪到最前面。
3.  **忍痛割爱**：如果箱子满了，你又想放新玩具，那就必须把那个**最久没玩过**的玩具（排在最后面的那个）给扔掉。

这就是 **LRU (Least Recently Used)**，即“最近最少使用”规则。

### 🪄 魔法双剑：哈希表 + 双向链表

为了让“找玩具”和“挪位置”都变得超级快，我们需要两件宝贝：

1.  **哈希表 (哈希快速找)**：
    *   它像是一个“索引卡片”，上面记录了每个玩具在箱子里的具体位置。
    *   有了它，我们一秒钟就能找到玩具，不用翻箱倒柜（时间复杂度 O(1)）。
2.  **双向链表 (链表管进出)**：
    *   它像是一串手拉手的小朋友。
    *   **头部长官**：负责迎接刚玩过的“新宠”。
    *   **尾部守卫**：盯着那个最久没玩的“倒霉蛋”。
    *   因为是双向的，我们可以很方便地把某个小朋友从队伍中间“拎出来”放到最前面。

### 💡 为什么这么做？
哈希表保证了“快”，双向链表保证了“有序”。两者结合，就像给玩具箱装上了自动传送带，既聪明又高效！
