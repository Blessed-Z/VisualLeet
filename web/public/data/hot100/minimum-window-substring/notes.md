# 76. 最小覆盖子串 (The Stamp Collector)

## 💡 深度解析
小朋友，想象你在商场走廊上走，走廊的地板上到处都是字母贴纸。
你有张任务清单 `t`（比如上面写着：A要1个，B要1个，C要1个）。
你要在走廊上划出一段**最短的区域**，确保这段区域里的贴纸能盖满你的任务清单。

### 核心策略：寻找最省地儿的集齐方案
1. **右边伸长 (探索)**：你不断向右走，把看到的字母都捡起来放进口袋。
2. **集齐瞬间**：当你发现口袋里的字母已经能盖满清单时，你停下来。
3. **左边缩进 (优化)**：你现在虽然集齐了，但可能划的地盘太大了。你尝试把左边界往右挪，看看丢掉一些字母后，口袋里是不是还能盖满清单。
4. **记录完美时刻**：每当缩到“再缩就盖不满了”的时候，你就记下这一段的长度。

这就是一个**“先贪心找全，再忍痛断舍离”**的过程！

---

## 🛠️ 执行步骤
1. **开清单**：把 `t` 的成分记在小本本上（`cnt_t`）。
2. **走右边**：`right` 每走一步，口袋 `cnt_s` 里的字母就多一个。
3. **查一查**：如果口袋里的字母 ≥ 清单里的（全都有且够数）：
   - 记录现在的宽度。
   - 尝试把 `left` 往右移（扔掉最左边的贴纸）。
   - 重复检查，直到口袋不够盖满为止。
4. **比大小**：最后选那个最短的。

---

## 📈 复杂度分析
- **时间复杂度**：$O(N)$。别看有个 `while`，每个字母最多也就被捡起来一次、扔掉一次。
- **空间复杂度**：$O(K)$。口袋里最多存 26 种字母。
