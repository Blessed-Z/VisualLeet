# 25. K 个一组翻转链表 (Group Reverse)

## 💡 深度解析
小朋友，想象有一列很长的小火车。老师说：“每 3 节车厢分成一组，让每组内的车厢调换方向（头变尾，尾变头），但如果最后剩下的一组不到 3 节，就不用动。”

### 核心策略：小组长带头调头
这道题就像是一个精细的工程：
1. **数人数**：我们先往后数，看看够不够 $k$ 个小朋友。
2. **定边界**：如果够了，我们就标出这一组的“组前人”和“组后人”。
3. **原地调头**：在“组前”和“组后”的看护下，让这 $k$ 个小朋友一个接一个地转身，反向拉手。
4. **归队**：翻转完这一组后，原来的组头变成了组尾，我们把它和下一组连接好。
5. **继续**：去数下一组。

如果最后人数不够了，我们就保持原样，直接收工！

---

## 🛠️ 执行步骤
1. 设置 `dummy` 节点。
2. 用 `tail` 指针往后数，每数到 $k$ 个就停下。
3. 调用 `reverse` 函数反转这一段。
4. `reverse` 函数逻辑：
   - 记录当前组的第一个节点（反转后会变成最后一个）。
   - 逐个调转指针方向。
   - 重新连接这组与前后的纽带。
5. 返回反转后的最后一个节点，作为下一组的起点。

---

## 📈 复杂度分析
- **时间复杂度**：$O(N)$。每个节点最多被数两次（数人数一次，反转一次）。
- **空间复杂度**：$O(1)$。我们只是在原地通过改变指针来调头。
