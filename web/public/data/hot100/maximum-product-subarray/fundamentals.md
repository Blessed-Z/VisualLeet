# 🧩 乘积最大子数组基础百科

### 🌟 核心概念：为什么不能只记录最大值？

在加法中，越加越大；但在乘法中，**负负得正**。
如果你现在的最大值是 `10`，最小值是 `-100`。
*   如果你遇到一个 `2`：最大变成 `20`。
*   如果你遇到一个 `-5`：最大值乘 `-5` 会变 `-50`，但**最小值乘 `-5` 会变 `500`**！
所以，之前的“谷底”才是翻盘的关键。

### 📚 变量小字典

| 变量名 | 比喻 | 作用 |
| :--- | :--- | :--- |
| `max_num` | **巅峰守护者** | 记录以当前元素结尾的连续子数组的最大乘积。 |
| `min_num` | **谷底守护者** | 记录以当前元素结尾的连续子数组的最小乘积。 |
| `ans` | **历史纪录片** | 记录到目前为止见过的所有最高峰。 |
| `nums[i]` | **当前的加速/反向点** | 数组中的下一个数字。 |

---

### ⚠️ 敲黑板

*   **连续子数组**：必须是连在一起的。
*   **乘 0 必归 0**：如果遇到 0，前面的所有累积都会消失，必须从下一个位置重来（或者以 0 为结果）。
*   **初始化**：通常初始化为数组的第一个数 `nums[0]`。
