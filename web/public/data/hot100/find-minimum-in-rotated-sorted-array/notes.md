# 📉 寻找谷底：旋转数组的秘密起点

小朋友，想象你正在爬一个环形的山坡。山坡本来是从低到高均匀上升的，但是有人把山坡切开，把高处的那一截平移到了低处的那一截的前面。

结果就是：现在的山坡会先升高，然后突然有一个极大的落差（断崖），掉到最低点，然后再继续升高。

我们的任务是：找出这个山坡上的**最低点**！

### 🪄 寻找秘诀：

我们要观察中点 `mid` 和右边界 `right` 之间的关系：

1. **观察中点 (`mid`) 和右边 (`right`)**：
   * 如果 `nums[mid] > nums[right]`：
     哎呀！中间的数字居然比最右边的还要大？这说明“断崖”一定在右半边！最低点一定在 `mid` 的右边。所以我们把左手 `left` 移到 `mid + 1`。
   * 如果 `nums[mid] <= nums[right]`：
     这说明从 `mid` 到 `right` 是一段平稳的上升路。那么最低点要么就是 `mid` 本身，要么在 `mid` 的左边。所以我们把右手 `right` 缩回到 `mid`。

2. **重复寻找**：
   直到两只手碰到一起，指着的那个地方就是全场最低的“谷底”啦！

### 💡 为什么不和 `left` 比较？
因为如果我们遇到 `[3, 4, 5, 1, 2]` 这种情况，`left` 比 `mid` 小，你无法判断断崖在左边还是右边。但是和 `right` 比较，目标方向总是非常明确的。
