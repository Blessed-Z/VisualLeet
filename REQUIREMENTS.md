# LeetCode AI Workspace - 核心需求文档 (V2.0 - 战略升级版)

> **声明**: 本文档包含项目的核心战略方向，所有标注为 [CRITICAL] 的项为最高优先级，禁止随意删除。

## 1. 战略目标
打造全球首款结合 **“硬核算法可视化”**、**“二次元沉浸式体验”** 与 **“社区共享知识库”** 的一站式刷题平台。通过技术与审美的极致融合，建立行业壁垒。

## 2. 核心功能模块 (按优先级排序)

### A. [CRITICAL] 数据库集成与优质内容共享 (Knowledge Base)
*   **技术选型**: Supabase (PostgreSQL) + Redis 缓存。
*   **核心逻辑**: 
    *   **持久化存储**: 所有 AI 生成的优质可视化动画、逻辑讲解和笔记必须存入云端。
    *   **零成本缓存**: 如果某道题目已有高质量回答，新用户请求时直接命中数据库，不调用 AI 接口，极大降低 API 运营成本。
    *   **社区互动**: 用户可以点赞、收藏他人公开的优质笔记。

### B. [CRITICAL] 二次元沉浸式体验 (Anime & Mascot)
*   **看板娘 (Mascot)**: 集成 Live2D 或 Spine 动画。在页面右下角增加一个“AI 助教”形象。
*   **状态联动**: 看板娘根据当前操作（思考中、成功、报错）做出不同的表情和动作。
*   **装饰元素**: 引入更多二次元风格的视觉符号、UI 装饰和音效。

### C. [HIGH] 浏览器插件化 (Browser Extension)
*   **无缝嵌入**: 支持作为 Chrome/Edge 插件运行，在 LeetCode 官网侧边栏直接唤起。
*   **上下文自动抓取**: 自动识别官网题目 URL，省去手动粘贴。

### D. [HIGH] 流式传输 (Streaming Response)
*   **即时反馈**: 逻辑讲解和 AI 对话必须支持逐字输出，提升体感速度。

## 3. 技术栈
*   **Frontend**: Next.js (App Router), Tailwind CSS v4, Monaco Editor.
*   **AI**: Universal Adapter (Gemini, Qwen, DeepSeek, OpenAI).
*   **Backend/Storage**: Supabase (Auth & DB).
*   **Animation**: PixiJS + Live2D.

## 4. 商业化路径
*   **MVP 阶段**: 验证二次元风格与可视化的用户反馈。
*   **增长阶段**: 通过社区共享内容降低单人成本，积累数据壁垒。
*   **变现阶段**: 点数制 (Credits) 或 订阅制，卖点是“极致的互动学习体验”。

## 5. [CRITICAL] 开源与授权策略 (Open Source Moat)
为了平衡“流量吸引”与“防范套壳”，采取以下策略：

### A. 开放内核 (Open Core)
*   **开源部分**: 核心 UI 框架、基础 AI 适配器逻辑。用于获取 GitHub Star、建立行业品牌、吸引社区贡献。
*   **闭源部分**: 核心 Prompt 工程、高级可视化生成算法、云端知识库数据、高级看板娘模型资产。

### B. 法律保护 (Licensing)
*   **开源协议**: 优先考虑 **AGPL-3.0** 或 **BSL (Business Source License)**。
*   **约束**: 严禁未经授权的商业化套壳。若基于本项目代码提供在线服务，必须开源其全站代码（AGPL）或在达到盈利阈值后付费（BSL）。

### C. 数据壁垒 (Data Moat)
*   **核心护城河**: 代码易抄，但 **Supabase 云端累积的优质题解数据**、**用户社交关系** 和 **二次元 IP 灵魂** 不可复制。
*   **官网优势**: 官方版本提供完整的云端同步、社区共享内容和最精良的 Live2D 交互体验，让用户选择“官网”而非“简陋的自建版”。
