# 239. 滑动窗口最大值 (King of the Window)

## 💡 深度解析
小朋友，想象有一群小朋友在排队量身高。我们要用一个**移动的相框**（窗口），每次圈住三个小朋友，找出这三个人里最高的那个。

### 核心策略：身高擂台赛
1. **大顶堆 (大王擂台)**：我们准备一个擂台，谁最高谁就站在最上面。为了方便，我们给每个小朋友发一个“号码牌”（索引），记录他是第几个。
2. **新队员入场**：每当相框向右移一步，就会有一个新的小朋友进入擂台。
3. **老队员退场 (延迟删除)**：如果现在擂台上最高的那个人，他的位置已经在相框外面了，说明他已经“过时”了，得请他下台。
4. **谁是英雄**：此时此刻站在擂台最高处的人，就是这个相框范围内的“身高大王”！

---

## 🛠️ 执行步骤
1. **搭建擂台**：把前 k 个小朋友的信息（身高取反、号码牌）放进堆里。
2. **记录首胜**：记下第一个窗口里的冠军。
3. **向前推进**：
   - 吞进右边的新小朋友：`heappush`。
   - 检查冠军是否过时：如果冠军的号码牌太小了，就 `heappop` 让他走。
   - 此时的冠军就是当前窗口的答案。
4. **循环往复**：直到相框滑到最后。

---

## 📈 复杂度分析
- **时间复杂度**：$O(N \log N)$。每次入堆和出堆的操作是 $\log N$，总共操作 $N$ 次。
- **空间复杂度**：$O(N)$。最坏的情况下，堆里可能会存下所有小朋友的信息。
